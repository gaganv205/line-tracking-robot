# Line Follower Robot вҖ” ROS 2 / Gazebo Simulation

A ROS 2 package that simulates a differential-drive robot capable of following a yellow line, recovering from broken-line gaps, and autonomously avoiding obstacles. The robot runs as a single state-machine node inside a Gazebo world and requires no manual control input.

### Key files explained

**`urdf/robot.urdf.xacro`** вҖ” Defines the physical robot:
- Rectangular blue chassis (0.3 Г— 0.2 Г— 0.1 m, 1.5 kg)
- Two driven wheels + one caster sphere
- Gazebo differential-drive plugin (publishes `/cmd_vel`, `/odom`)
- Downward-facing RGB camera (`/camera/image_raw`)
- 360В° LIDAR sensor (`/scan`)

**`worlds/line_world.world`** вҖ” A 20 Г— 20 m white ground plane with a yellow painted line track for the robot to follow.

**`launch/sim.launch.py`** вҖ” Starts four processes in sequence:
1. Gazebo with `line_world.world`
2. `robot_state_publisher` (URDF вҶ’ TF)
3. `spawn_entity.py` (places robot at origin)
4. `line_follower_node` (the control node)

**`line_follower_robot/line_follower_node.py`** вҖ” The autonomous controller (see [How It Works](#how-it-works)).

## Installation & Build

### 1. Set up your ROS 2 workspace

```bash
mkdir -p ~/ros2_ws/src
cd ~/ros2_ws/src
```

### 2. Clone the repository

```bash
git clone https://github.com/gaganv205/line-tracking-robot.git
```

This places the package directly inside `src/`:

```
~/ros2_ws/src/line-tracking-robot/

```

### 3. Build with colcon

```bash
cd ~/ros2_ws
source /opt/ros/humble/setup.bash
colcon build --packages-select line_follower_robot
```

### 4. Source the install overlay

```bash
source ~/ros2_ws/install/setup.bash
```

> **Tip:** Add the two `source` lines above to your `~/.bashrc` so you don't need to run them every session.

## Running the Simulation

Launch everything (Gazebo + robot + controller) with a single command:

```bash
ros2 launch line_follower_robot sim.launch.py
```

Gazebo will open with the line-world environment and the robot will spawn at the origin. The `line_follower_node` starts immediately and begins autonomous operation вҖ” no additional commands are needed.

### Watching ROS topics (optional)

Open a second terminal, source the workspace, then:

```bash
# View velocity commands being sent to the robot
ros2 topic echo /cmd_vel

# View raw LIDAR readings
ros2 topic echo /scan

# List all active topics
ros2 topic list
```

### Stopping the simulation

Press `Ctrl+C` in the launch terminal. Gazebo will close along with all nodes.

## How It Works

The node implements a three-state finite state machine:

```
         line lost                    obstacle detected
FOLLOWING в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ> SEARCHING      в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ> AVOIDING
    ^                     |                                   |
    в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ                                   |
          line found again            obstacle cleared +       |
                                      line found              |
    ^в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```

### State: `FOLLOWING`
- Captures each camera frame and converts it to HSV colour space.
- Masks yellow pixels (HSV range `[20,100,100]` вҖ“ `[35,255,255]`) in the lower half of the frame (ROI).
- Finds the largest contour; computes its centroid horizontal error relative to the image centre.
- Applies proportional control (`Kp = 0.010`) to steer while driving forward at 0.2 m/s.
- Switches to `SEARCHING` if the line is lost for longer than the grace period.
- Switches to `AVOIDING` if any LIDAR reading within the front arc is < 0.8 m.

### State: `SEARCHING`
- Robot turns in the last-known line direction at 0.5 rad/s.
- Alternates turn direction after `MAX_SEARCH_ITERS` (60) iterations without finding the line.
- Returns to `FOLLOWING` as soon as the yellow line is detected again.

### State: `AVOIDING`
Runs a multi-phase avoidance manoeuvre:
1. **stop_and_back** вҖ” halt and reverse briefly.
2. **turn_right** вҖ” rotate right to clear the obstacle.
3. **forward** вҖ” drive forward past the obstacle.
4. **turn_left** вҖ” rotate left back toward the line.
5. **move_to_line** вҖ” creep forward to reacquire the line.
6. **turn_left_final** вҖ” final alignment turn.

Returns to `FOLLOWING` once the obstacle is clear and the yellow line is detected.

## Tunable Parameters

All constants are defined at the top of `LineFollowerNode` in `line_follower_robot/line_follower_node.py`:

| Constant | Default | Description |
|---|---|---|
| `YELLOW_LOWER_HSV` | `[20, 100, 100]` | Lower bound of yellow in HSV |
| `YELLOW_UPPER_HSV` | `[35, 255, 255]` | Upper bound of yellow in HSV |
| `LINE_MIN_AREA` | `500` pxВІ | Minimum contour area to count as a line |
| `ROI_TOP_FRAC` | `0.5` | Fraction of frame height used as ROI (bottom half) |
| `FOLLOW_SPEED` | `0.2` m/s | Linear speed while line-following |
| `SEARCH_SPEED` | `0.5` rad/s | Angular speed while searching |
| `KP` | `0.010` | Proportional gain for steering |
| `OBSTACLE_DIST` | `0.8` m | Distance at which obstacle avoidance triggers |
| `AVOID_CLEAR_DIST` | `1.0` m | Distance considered "obstacle cleared" |
| `MAX_SEARCH_ITERS` | `60` | Iterations before reversing search direction |
| `SEARCH_FORWARD_STEPS` | `60` | Forward steps taken at the start of a search |

## Troubleshooting

**Gazebo doesn't open / crashes immediately**
- Ensure `gazebo_ros` is installed: `sudo apt install ros-humble-gazebo-ros-pkgs`
- Check that `GAZEBO_MODEL_PATH` and `GAZEBO_RESOURCE_PATH` are set correctly after sourcing ROS.

**Robot doesn't move**
- Verify the node is running: `ros2 node list` should show `/line_follower_node`.
- Check the camera topic is publishing: `ros2 topic hz /camera/image_raw`.

**`xacro` not found during launch**
- Install it: `sudo apt install python3-xacro`

**Line not detected**
- The simulation world uses a yellow line. If you modify the world, adjust `YELLOW_LOWER_HSV` / `YELLOW_UPPER_HSV` to match your line colour.
- Increase `LINE_MIN_AREA` if noise is causing false positives.

**Package not found after build**
- Make sure you sourced the install overlay: `source ~/ros2_ws/install/setup.bash`
- Rebuild and re-source if you made changes: `colcon build --packages-select line_follower_robot && source install/setup.bash`

**`colcon build` fails with "package not found"**
- Confirm `package.xml` is at the repository root (same level as `setup.py`), not inside a subdirectory.
